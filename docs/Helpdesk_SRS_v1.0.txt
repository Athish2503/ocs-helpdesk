Introduction1.1 PurposeThis Software Requirements Specification (SRS) defines the functional and non-functional requirements for an AI-Powered Helpdesk System. The system enables organizations to manage customer support tickets efficiently by leveraging artificial intelligence for automated responses, smart routing, and a knowledge base with Retrieval-Augmented Generation (RAG).This document is intended for developers, system architects, QA engineers, project managers, and stakeholders involved in the design, development, testing, and deployment of the system.1.2 ScopeThe AI-Powered Helpdesk System is a full-stack web application that includes:A customer-facing ticket submission and chat interfaceAn AI assistant powered by Google Gemini that answers queries from a knowledge base before escalating to human agentsA support agent dashboard for ticket management, communication, and resolutionAn admin panel for user management, analytics, and system configurationA knowledge base management module for articles and embeddingsEmail notification system for ticket lifecycle eventsAudit logging and compliance tracking1.3 Overall Description2.1 Product PerspectiveThe AI-Powered Helpdesk System is a new, standalone product designed to replace or augment traditional manual helpdesk workflows. It integrates with external email relay services and the Google AI Studio API. The system is deployed on a single VPS initially, with architecture designed for horizontal scaling when user load demands it.2.2 Product FunctionsAt a high level, the system provides:AI-first ticket deflection — the system attempts to resolve user queries automatically using the knowledge base before creating a ticketTicket lifecycle management — creation, assignment, escalation, resolution, and closureKnowledge base authoring and embedding — articles are stored with vector embeddings for semantic searchReal-time agent-customer communication via WebSocketsEmail notifications for all ticket lifecycle eventsAnalytics and reporting dashboard for managementAudit logging for compliance and debugging2.3 User Classes and CharacteristicsUser ClassDescriptionTechnical LevelAccess LevelEnd User / CustomerSubmits support requests via portal or chat widgetLowPublic + authenticated portalSupport AgentHandles tickets, communicates with users, resolves issuesMediumAgent dashboardSupervisor / Team LeadMonitors agent performance, escalates, views analyticsMedium-HighAgent dashboard + analyticsAdminManages users, configures the system, manages KB, views all audit logsHighFull admin panelSystem (AI Agent)Automated responses, embedding generation, ticket classificationN/AInternal API2.4 Operating EnvironmentThe system operates in the following environment:Server: Ubuntu 24 LTS on a VPS with Docker Compose orchestrationBrowser Support: Chrome 100+, Firefox 100+, Safari 15+, Edge 100+ (modern evergreen browsers)Mobile: Responsive web UI; native apps not in scope for v1Network: Public internet access via Cloudflare CDN and DNSAI Provider: Google AI Studio (Gemini 1.5 Pro / Flash) via REST API2.5 Design and Implementation ConstraintsAll services must be containerized using Docker and orchestrated via Docker ComposeNo Kubernetes in Phase 1; architecture must support migration to K8s in Phase 2All sensitive configuration (API keys, DB passwords) must use environment variables; no hardcoded secretsThe AI layer must be abstracted behind an internal service interface to allow provider swappingFile uploads must be stored in MinIO (S3-compatible); no direct filesystem uploads in productionAll database access must go through Prisma ORM; no raw SQL in application logic except for performance-critical vector search2.6 Assumptions and DependenciesGoogle AI Studio API will remain available and pricing will not change significantly during MVPVPS provider guarantees 99.5% uptime SLAEmail relay (Brevo or Resend) will maintain free tier for initial volumepgvector extension is available on the PostgreSQL installationCloudflare free plan is sufficient for DDoS protection and CDN at MVP scale3.
